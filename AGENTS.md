# mikemcquaid.com — Knowledge Base

This repository is Mike McQuaid's personal website and serves as a
centralised knowledge base of his public statements, writings, talks,
interviews and thoughts.

## Who is Mike McQuaid?

See `now.md` for what he's currently doing and `cv.md` for his full
career history.

## Content locations

### Solo content (Mike's words only)

- **Articles** (`_posts/`) — Long-form blog posts. Markdown and
  AsciiDoc. Mike's most considered, authoritative public statements.
- **Talks** (`_talks/`) — Conference presentations. The entire
  transcript is Mike speaking solo. Front matter has `youtube-id`
  and/or `speakerdeck-id`.
- **Thoughts** (`_thoughts/`) — Short-form posts. Body text is Mike's;
  `link_title`/`link_description` metadata is from the linked page.

### Group content (Mike + others)

- **Interviews** (`_interviews/`) — Conversations with journalists,
  podcast hosts and interviewers. Contains both Mike's words and the
  interviewer's. Front matter has `publication`, `link`, and optionally
  `youtube-id` or `podcast_audio_url`.
- **External podcasts** (`_data/podcasts_external.yml`) — Episodes from
  podcasts Mike co-hosts (Minimum Viable Management, Balancing Dads).
  Multiple speakers. Feed sources in `_data/podcast_sources.yml`.

### Transcripts (`_data/transcripts/`)

YAML files mapping integer timestamps (seconds) to text segments.
Generated by Whisper from video/audio. Format:
```yaml
0: First segment of speech...
9: Next segment of speech...
```
To find the source, search `_talks/` and `_interviews/` for files with
matching `youtube-id`, `transcript-id`, or filename stem.

## How to query this knowledge base

### Finding Mike's views on a topic
1. Search `_posts/` first (most authoritative, solo)
2. Search `_thoughts/` for short-form opinions (solo)
3. Search `_data/transcripts/` for spoken statements
4. Cross-reference transcript IDs with `_talks/` (solo) or
   `_interviews/` (group) for context

### Reading content
Read the markdown file directly. Content is after the `---` front
matter delimiters.

## Repository structure

```
_posts/           Articles (solo, Markdown/AsciiDoc)
_talks/           Talk metadata + content (solo)
_thoughts/        Short-form thoughts (solo)
_interviews/      Interview metadata + content (group)
_data/
  transcripts/    Whisper transcripts (YAML)
  podcasts_external.yml  External podcast episodes (group)
  podcast_sources.yml    Podcast RSS feed URLs
images/media/     Locally cached media (youtube/, talks/, thoughts/, interviews/)
lib/              Ruby scripts for enrichment and transcription
```
